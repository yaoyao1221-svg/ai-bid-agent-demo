# AI 标书精灵 Demo

这个 demo 用于验证“上传招标文件后，系统能准确分析并生成投标材料”的核心闭环，不用于宣传页展示。

## 当前已跑通的链路

```text
企业本体样例
+ 招标文件上传
→ 招标要求解析
→ 企业适配度评分
→ 风险与缺口报告
→ 标书草稿生成
→ 审查 Agent 校验
→ 分析过程说明
```

## 运行方式

```bash
cd demo
npm test
```

浏览器查看：

```bash
cd demo
python3 -m http.server 5173
```

然后访问 `http://localhost:5173`。

如果浏览器允许直接加载本地 ES module，也可以直接打开 `index.html`。

## 招标文件上传

当前 demo 页面提供“上传招标文件”按钮和“开始解析”按钮。

已支持：

- `.txt`
- `.md`
- `.json`
- `.doc`
- `.docx`

暂不直接解析：

- `.pdf`

`.docx` 已支持基础正文抽取；`.doc` 会尝试从旧版 Word 二进制中抽取可读正文。如果 Word 文件里主要是扫描图片、加密内容或复杂版式，仍需要另存为 `.docx` 或后续接入 OCR。

## 企业信息输入

当前 demo 支持两种方式维护企业资料：

1. 页面左侧手动填写。
2. 上传 JSON 格式企业资料。

手动填写格式：

企业资质，一行一条：

```text
资质名称|等级|有效期
营业执照|企业基础资格|2099-12-31
```

企业案例，一行一条：

```text
项目名称|行业|金额|年份|标签1,标签2
重庆市教育矫治局办公终端采购响应项目|政府采购/办公终端设备|54240|2024|办公终端,华为擎云C7,政府采购
```

企业 JSON 示例：

```json
{
  "name": "重庆科讯科技有限公司",
  "slogan": "办公终端、信息化设备供应与本地化服务供应商",
  "qualifications": [
    { "name": "营业执照", "level": "企业基础资格", "validTo": "2099-12-31" }
  ],
  "cases": [
    {
      "name": "重庆市教育矫治局办公终端采购响应项目",
      "industry": "政府采购/办公终端设备",
      "amount": 54240,
      "year": 2024,
      "tags": ["办公终端", "华为擎云C7", "HarmonyOS", "设备供货", "原厂质保", "政府采购"]
    }
  ],
  "capabilities": ["办公终端设备供货", "华为擎云C7设备响应", "HarmonyOS办公终端配置", "设备参数响应"],
  "service": {
    "localOffice": true,
    "responseHours": 24
  }
}
```

当前科讯样例还额外保留了营业执照状态、设备参数、完税证明状态、资产负债表状态和待补证据清单。这些字段来自企业资料抽取结果，只作为“平台已读懂/仍缺什么”的依据；凡是缺少 OCR 或人工确认的字段，不会被当作确定事实参与高置信匹配。

后续补全企业资料时，按项目内 `skills/enterprise-profile-extractor/SKILL.md` 的流程执行：文件切片、分类抽取、证据绑定、保守校验、本体入库。

## 准确性原则

1. 硬性资格缺失时，系统不得推荐参与。
2. 企业事实必须来自企业本体样例，不允许凭空生成资质、案例、金额或响应能力。
3. 部分满足和低置信度结果必须进入风险或人工确认。
4. 标书草稿只能作为待复核材料，不能直接投递。
5. 审查链路需要把硬性缺口标为高风险，把低分风险标为中风险。
6. 分析过程必须用客户能理解的语言说明依据、结论和下一步动作。

## 当前刻意不做的内容

- 不做真实 PDF/OCR。
- 不接真实大模型。
- 不做自动抓取招标信息。
- 不做自动投递。
- 不做真实胜率预测。
- 不做复杂报价模型。

这些能力后续应在当前准确性闭环稳定后逐步接入。
