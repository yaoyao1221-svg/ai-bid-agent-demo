# AI 标书精灵 Demo

这个 demo 用于验证“上传招标文件后，系统能准确分析并生成投标材料”的核心闭环，不用于宣传页展示。

## 当前已跑通的链路

```text
企业本体样例
+ 招标文件上传
→ 招标要求解析
→ 企业适配度评分
→ 风险与缺口报告
→ 标书草稿生成
→ 审查 Agent 校验
→ 白盒推理节点展示
```

## 运行方式

```bash
cd demo
npm test
```

浏览器查看：

```bash
cd demo
python3 -m http.server 5173
```

然后访问 `http://localhost:5173`。

如果浏览器允许直接加载本地 ES module，也可以直接打开 `index.html`。

## 招标文件上传

当前 demo 页面提供“上传招标文件”按钮和“开始解析”按钮。

已支持：

- `.txt`
- `.md`
- `.json`

暂不直接解析：

- `.pdf`
- `.doc`
- `.docx`

这些格式需要后续接入 OCR、版式解析、表格抽取和 Word 文档解析服务后再开放，demo 不会假装已经可以精准解析。

## 企业信息输入

当前 demo 支持两种方式维护企业资料：

1. 页面左侧手动填写。
2. 上传 JSON 格式企业资料。

手动填写格式：

企业资质，一行一条：

```text
资质名称|等级|有效期
ISO9001质量管理体系认证|认证|2028-12-31
```

企业案例，一行一条：

```text
项目名称|行业|金额|年份|标签1,标签2
重庆智慧园区运维平台项目|信息化|5800000|2024|系统集成,平台运维,本地服务
```

企业 JSON 示例：

```json
{
  "name": "重庆云桥数字科技有限公司",
  "slogan": "面向政企客户的系统集成、数据治理与平台运维服务商",
  "qualifications": [
    { "name": "ISO9001质量管理体系认证", "level": "认证", "validTo": "2028-12-31" }
  ],
  "cases": [
    {
      "name": "重庆智慧园区运维平台项目",
      "industry": "信息化",
      "amount": 5800000,
      "year": 2024,
      "tags": ["系统集成", "平台运维", "本地服务"]
    }
  ],
  "capabilities": ["系统集成", "数据治理", "平台运维", "本地化售后"],
  "service": {
    "localOffice": true,
    "responseHours": 4
  }
}
```

## 准确性原则

1. 硬性资格缺失时，系统不得推荐参与。
2. 企业事实必须来自企业本体样例，不允许凭空生成资质、案例、金额或响应能力。
3. 部分满足和低置信度结果必须进入风险或人工确认。
4. 标书草稿只能作为待复核材料，不能直接投递。
5. 审查链路需要把硬性缺口标为高风险，把低分风险标为中风险。

## 当前刻意不做的内容

- 不做真实 PDF/OCR。
- 不接真实大模型。
- 不做自动抓取招标信息。
- 不做自动投递。
- 不做真实胜率预测。
- 不做复杂报价模型。

这些能力后续应在当前准确性闭环稳定后逐步接入。
